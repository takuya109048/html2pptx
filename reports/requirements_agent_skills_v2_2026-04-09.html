3. 結論（要件定義支援の可否）

  • 結論A: 要件定義を直接支援するAgent Skillは公開実例として存在する（requirements-elicitation / requirements-analysis）。
  • 結論B: 主要基盤（Claude/OpenAI/Cursor/LangChain）は、専用Skillがなくても「MCP + Jira/Confluence連携」で要件定義ワークフローを実装可能。
  • 結論C: GitHub Copilotは方式自体は強力だが、GitHub App型Extensionsの終了（2025-11-10）を前提に、MCP/VS Code側へ設計を寄せる必要がある。
