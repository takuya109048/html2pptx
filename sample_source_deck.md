# 長期文脈は設計と運用で安定する
## CLAUDE.mdは「AIに何度も同じことを教えない」ための仕組みである

| key | value |
|-----|-------|
| layout | table_conclusion |
| note | この最終スライドで伝えたいことは、CLAUDE.mdの価値はファイルを置くこと自体ではなく、継続的にAIの判断を整えることにあるという点である。ルールを厳選し、具体的に書き、スコープを分け、運用の中で更新する。この流れがそろうと、AIは毎回の会話に依存せず、プロジェクトに合った振る舞いを再現しやすくなる。したがって、CLAUDE.mdは長期文脈の設計図である。 |

```table
| 設計観点 | 実務上の判断 | 期待される効果 |
| --- | --- | --- |
| **内容** | 常に守るルールだけを残す | 重要指示が埋もれない |
| **表現** | 1文、宣言調、実行可能に書く | AIが行動へ変換しやすい |
| **配置** | グローバル、プロジェクト、ディレクトリで分ける | 文脈ノイズが減る |
| **運用** | 二度言った注意を追加し、古い規約を削る | 現場に合わせて精度が上がる |
```
```conclusion
**CLAUDE.mdは、AIの長期文脈をプロジェクト資産へ変えるためのルール設計である。**
```
