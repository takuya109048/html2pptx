# CLAUDE.mdとは何か
## セッション開始時に必ず読み込まれる「ルールブック」

| key | value |
|-----|-------|
| layout | plain_image_row |
| image_label_1 | CLAUDE.mdがClaudeセッション開始時に自動読み込みされるルールブックとして機能する仕組みの横長インフォグラフィック。フラットデザイン、白背景、文字なし。16:5の横長比率で「ユーザーがルールを書く→CLAUDE.mdに保存→セッション開始時に自動読み込み→Claudeが適切に振る舞う」の4ステップを矢印でつないで図解する。 |
| note | CLAUDE.mdは「設定ファイル」ではなく「ルールブック」という位置づけが重要。3大ゴール: ①重要ルールの厳選（常に守るものだけ）②適切な密度（60〜300行）③スコープの分離（ディレクトリごとに役割分担）。 |

```card-a
### CLAUDE.mdの3大設計ゴール
- **重要ルールの厳選**: 常に守ってほしいルールのみを記述する（単発の指示は会話で行う）
- **適切な密度**: 60〜300行に抑え、重要なルールが埋もれないようにする
- **スコープの分離**: プロジェクト・ディレクトリごとに役割を分け、AIへのノイズを減らす
```

# 良いルールの3条件
## 曖昧な指示はAIの精度を下げる

| key | value |
|-----|-------|
| layout | list_3card |
| note | 良いCLAUDE.mdのルールには3つの共通点がある。NG例「テストはしっかり行うこと」→OK例「変更後は pnpm test を実行し、失敗したテストをすべて修正する」。[nanobanana2 icon prompt] 3つのフラットアイコンを横一列に並べた1枚の画像を生成する。左から順に: アイコン1: シンプルな1行テキスト・短い箇条書きのシンボル アイコン2: 力強いOKチェックマーク・断言・確定のシンボル アイコン3: ターミナル・コマンド実行・コードのシンボル。各アイコンは正方形の枠内に収まるシンプルなフラットデザイン。白背景、青系色統一、ミニマルスタイル、文字なし。※生成後に水平3等分にトリミングして各カードのアイコンとして使用。 |

```card-a

```

```card-b

- 断定的な表現がAIへの指示として機能しやすい
- **OK:** 「変更後は `pnpm test` を実行する」
```

```card-c

- 抽象的なスローガンを避ける
```
