| note | Claude Codeには長期文脈を維持するための2つの仕組みがあります。CLAUDE.mdはユーザー自身が記述するルールブックで、セッション開始時に必ず読み込まれます。MEMORY.mdはClaudeが自動で書き込む学習メモです。この2つを使い分けることで、毎回同じ説明を繰り返す手間が省けます。 |

```card-a
**CLAUDE.md** はユーザーが書く「ルール・前提・手順」の指示ファイル。セッション開始時に必ず読み込まれ、AIの振る舞いをプロジェクトの基準に完全に適合させる。

**MEMORY.md** はClaudeが自動で書き込む「学習したパターンと好み」のメモ。意識せずとも使うほどに賢くなる仕組みで、明示的に管理しなくてもよい。
```

# 長さ・情報量のベストプラクティス
## 精度を下げる「過多」と「曖昧」を避ける

| key | value |
|-----|-------|
| layout | plain_image_col |
| image_label_1 | ルール記述のベストプラクティスを表すシンプルなイラスト。フラットデザイン、ミニマルアイコン風、ビジネス向け、白背景。6:5の比率で「精密で簡潔な指示書」の概念を1枚のビジュアルで表現する。文字なし。 |
| note | CLAUDE.mdの推奨サイズは60〜300行です。良いルールは1ルール1文で短く具体的に書き、宣言調で断定します。「テストをしっかり行う」ではなく「pnpm testを実行し、失敗したテストをすべて修正する」のように実行可能な指示にすることが精度向上の鍵です。 |

```card-a
### サイズと書き方の3原則
**推奨サイズ**: 60〜300行（2〜6画面程度）。Claudeが安定して遵守できるルール数は150〜200程度が上限。

良い `CLAUDE.md` のルール記述には3つの共通点がある。

1. **1ルール1文**: 短く、具体的に書く
2. **宣言調**: 「〜する」と断定する（「してください」は使わない）
3. **実行可能**: 行動レベルで指定する。「テストをしっかりやれ」より「`pnpm test`を実行して失敗を修正せよ」
```
